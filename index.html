Augmented Reality App

Throughout the module we looked into a few art practises such as GPS drawing, psychogeopgraphy and music concrète. We now have been given the assignment to create an augmented reality (AR) app incorporating one of the art practises. After a bit of tinkering I came up with a few ideas and elements I could use for an app.

Some ideas mapped out, which might or might not make sense to other people :) I’ll cover the two most interesting ideas I had.

The first one is a bit of a mix between the practises. The idea evolved around (live) image manipulation, influenced by the user input about the feeling of the area or by sound input. The user could give an input (maybe the user creates sound as input) or soundbites from the surrounding would be used for input which would be used as paramaters for an image manipulation algorithm. Besides the standaard filter effects for photo manipulation I was thinking about applying algorithms which would have a glitch effect or something like pixel sorting.

Pixel sorting is something I recently discovered and would like to play around with some day. Pixel sorting is the ‘invention’ of Kim Asendorf. It would be interesting to see if this could be used to express emotions and feelings, or give the image of a certain place a certain atmosphere.

An example of a pixel-sorted photo by Kim Asendorf

The second idea had more to do with GPS drawing. The idea is to make it crowdsourced instead of drawing it on your own. The goal is to make it a group activity, which also means there has to be a ‘commitment’ as a group to create a drawing. This idea has been inspired by the ‘Do Not Touch’, interactive & crowdsources video where users ‘participate’ in the video with their mouse cursos. The cursor is also being recorded and added to the video.

A part of the ‘Do Not Touch’ video where the users have to create a smiley.

The interesting part about crowdsources artwork is that individual choices of everybody influence the collective outcome. This also reflects how we live together as humans in communities. We all make our own individual choices, but everything has impact on all the other individuals, direct or indirect and the collective whole. But we won’t get to philosofical now :).

I struggled a bit with the AR implementation , the initial ideas I had didn’t really make use of any augmented reality features. So the requirement of AR has been the reason why I had to drop quite a few of ideas. The 0ne idea I could fit with AR was the crowdsourced GPS drawing, so that’s the concept I picked to build a prototype for.

The goal of the app is to create a collaborative experience, make people interact with each other and work towards a collective goal. In the process of this people will also interact and be influenced by the terrain and environment, as with the original form of GPS drawing.

As you can see the app itself consists of the camera view and a (ugly) button to add a new point to the data set. The image target is being displayed in front of the camera, where you can see some testdata I’ve put into the database. It’s supposed to look like a question mark, I might have forgot a few points :-P The green dots it the data set being visualised, the red dot is the current position (when available).

This is a screenshot of the app in function. You can see how they display the dots.

Technical Implementation

THe prototype consists of two parts; the app for android itself and a server application. The android app is build with Unity3D and Vuforia, an augmented reailty toolkit. Developing in Unity3D has been a bit of a jump in at the deep end with 0 experience in either Unity or C#.

On the server side I have written a Node.js app linked with a PostgreSQL, all running on Heroku. For this purpose websockets would have been a good way for the client/server communication but after looking into it I found it was too complicated to implement in the client side (at least for someone without any C# experience :) ) and because I was building a prototype where performance isn’t the biggest priority I decided to go with an easier solution. The server has routes for two url’s; ending with /db and /pos/(lat),(long).

The first route queries the database and pulls all the locations out of the database and returns a webpage with the data in JSON. With the second URL the latitude and longitude coordinates are included into the URL. The server grabs this as a string, splits it and inserts it into the database.

In Unity the request go through a WWW-object, which is used to load the content of a webpage. When requesting the locations, it receives the JSON in the webpage which is then further parsed in Unity. When submitting data it doesn’t receive any data, it is just being used to send the request to the server, which makes the whole implementation of the server/client a lot simpler than using websockets.

After the JSON is parsed it processes the values, scales them accordingly to fit the image target and is then displayed with dots.

Options for expansion/improvement

It’s still a prototype, there are quite a lot of things which could be added or improved (besides more prettyness in the UI ed).

  • If this would become a real app it wouldn’t be very easy to use if it displays everything in one big map (but if it was, it might also look quite interesting? Or display all the drawings as a big drawing wall?). Probably there needs to be a feature which enables you to select different ‘projects’.
  • It would be cool if you could pick unique image targets for your drawing/project, improving the relation between the environment and the artwork
  • The server doesn’t work very effective with polling everytime. A better technology like websockets might be a good improvement
  • Include altitude into data so the data becomes more of a point cloud

Public Artworks

The Monument to the Conquerors of Space

The Monument to the Conquerors of Space (1964) is a monument in Moscow build in honour of Russia’s preeminence in space. In 1961 Yuri Gagarin made the first orbit around the earth and went to outer space. The monument consists of a big obelisk with a curve (110 meters high) made with titanium, with on top a rocket. The monument is to resemble the rocket and the plumes created when taking off. On the base of the monument are engravings picturing resembling the people involved in the space programm.

The purpose of te Monument to the Conquerors of Space is to celebrate the achievements in space made by the Soviet Union. In the 60’s the Cold War was in full swing and the Soviet Union had beaten the USA in making the first steps into space. The monument was the symbolisation of these victories in the space race and functions to pride the Soviet Union achievements and celebrate the greatness of the Union.

The location of the monument might have been inspired by the fact that most parts of the space programm were based in these parts of the city. Korolyov, the lead engineer, lived a few blocks away.

The monument is a permanent building and also houses the Memorial Museum of Cosmonautics. The Soviet Union build the monument, so there was no lack of authorisation. It even seems they started planning the monument 3 years before Gagarin made his orbit, which points out how important it must have seem to be a step ahead of the USA and basically ‘show off’, especially seen in the light of the Cold War and the Arms & Space Race going on.

Vectorial Elevation

Vectorial Elevation is a piece consisting of robitic search lights, which were directed by participants on the internet. The first time the work was shown was in 1999 in Mexico City to celebrat the arrival of the year 2000. It has been repeated on several locations and occasions, such as in Vancouver for the Winter Olympics in 2010.

One of the unique aspects of this work is the scale and interaction with the public. By going to a website, everybody can create a lightsculpture and submit it to the queue, which is quite a lot of control for one of the biggest interactive artworks made.

By not running a preprogrammed show, but give the public control it creates a sence of participation.

The piece has been designed to ‘attract personal participation that creates a sense of connection, conplicity and entitlement’. By not being merely spectators people feel that they are part of something bigger and have a place.

The locations of the work are in a central place, closely or on location of the festivity and viewable from the surroundings (depending on the strength of the lights it can be seen from 15 km).

The work is a temporary installation, commissioned by organizations involved in the respecive festivity.

If you compare the works they’re quite different, both in forn and functionality. The Monument to the Conquerers of Space is a symbol of national pride and achievement to impress people while the Vectorial Elevation is made to make people feel part of it, give them a place in society.

GPS drawing

This week we talked about GPS drawings. This is the practise of walking a figure and recording your GPS locations so these can be digitally visualised into your ‘drawing’.

One of the characteristics of a GPS drawing is the influence the geoography has on your drawing. Because of (articifial or natural) obstacles forces you are forced into certain paths or limited in the choice of directions, which gives that every drawing has a ‘signature’ of the location where it was drawn.

The funny thing is that this, depending on the area, can be mostly influenced by the choices of other people. With environments mostly modified or created by humans you don’t run into these obstacles if you follow the purpose of these structures. But when diverting from this ‘set path’, the structure that was first ment as for example an improvement has become an obstruction or vice versa.

To me the appleal of GPS drawing seems to be in the fact that you can see it as a struggle between you and the location. Humans have always had a tendency to leave a trail and shape their environment to their benefits, but it will never suit all purposes and goals. When going on a walk (or other means of transportation) you are confronted with these elements and are forced to be creative with it.

Music Concrète - The music surrounding us

Yesterday Mike gave a presentation for the module DAT201 (Strageties for Art & Technology) about Sound Art, covering several topics and talked about some of the questions. One of the questions was about noise: ‘Is noise music?’. That question goes back to the question ‘What is noise?’.

Today’s session was a perfect continuation of this topic. We talked about environmental sound with the goal to be more aware of the sounds surrounding you and appriciative of its characteristics.

Let’s walk together through a great modern capital, with the ear more attentive than the eye, and we will vary the pleasures of our sensibilities by distinguishing among the gurglings of water, air and gas inside metallic pipes, the rumblings and rattlings of engines breathing with obvious animal spirits, the rising and falling of pistons, the stridency of mechanical saws, the loud jumping of trolleys on their rails, the snapping of whips, the whipping of flags. We will have fun imagining our orchestration of department stores’ sliding doors, the hubbub of the crowds, the different roars of railroad stations, iron foundries, textile mills, printing houses, power plants and subways.

Luigi Russolo, The Art of Noise, 1913

There is a lot of stuff happening around you, but brains are really good at filtering away the ‘backgroundnoise’ to help you focus.

We went out and walked around campus with an audiorecorder listening to the sound around us and trying to capture some of those sounds. Quite a lot of sound is too soft or far away to clearly make a record of. But we were able to record some interesting sounds in the workshop, UPSU, outside buildings and from a watertap.

Then we imported the sounds into Audacity and played around with them. I played around with several effects, among which the ‘Paulstretch’. By stretching the audio out, you can get some really nice ambient sounds. I’ve also done some other things and created a small composition with the different sounds.

An interesting app I came across is ‘Recho’. It lets you record soundbites and publish them. But to listen to the recording you need to be in the same location. This makes the recording in something bigger than just a soundrecording, but more of a memory or reference to something that has happened.

Psychogeography - 'Falling in love with places'

In the first practical session of Reflexive Design Mike talked about ‘Psychogeography’, which is defined as ‘the study of the precise laws and specific effects of the geographical environment, consciously organized or not, on the emotions and the behavior of individuals’. Practically that means making geographical distinction between areas based on emotions.

The emotional judgement of locations is something I recognise doing. Two weeks ago I moved from the Netherlands to Plymouth, so I have loads of stuff to discover here. In the first week you get a lot of impressions and feelings when walking through Plymouth. But I’ve also noticed that in a short period as a week or two things start to feel normal and you pay less attention to your surroundings and how it makes you feel.

As noted by the worksheet the main practise to do psychogeography is ‘derive’: ‘an aimless wandering with senses open to how certain areas make you feel’. We will be using a slightly different approach, We’ll draw a circle on the map and plan a route that follows the circle as close as possible.

Because I just moved to Plymouth it was a bit difficult to follow the assignment strictly and do an imaginary walk and divide the map up in area’s that have different feelings. So we went straight to the real walk part!
